{# Tag-suggestion review modal. Opened by the browse-page "Review tags…" button after the /api/maintain/suggest-tags-review job finishes. The toast's onDone callback hands the job_id to window.makiTagReview.open(jobId), which fetches the inverted `tag → candidates` payload from /api/jobs/{jobId}/result and renders this two-pane layout: - Left: tag list sorted by candidate count, with a filter input and per-row count + mean-confidence bar. Clicking a row picks it as the active tag, the right pane renders its candidates. - Right: thumbnail grid of candidate assets for the active tag, each pre-checked with confidence overlay. Threshold slider auto-unchecks below X; "skip already-tagged" hides assets that already carry the tag. "Apply " hits the existing POST /api/batch/tags with the selected asset_ids + [tag]. On success the row strikes through and the next un-applied row is selected. "Skip" advances without applying. Mounted from base.html alongside the maintain dialog so any page can re-attach to a finished review job (page-reload survival). The result payload lives on Job.result in-memory; a server restart drops it. #}
